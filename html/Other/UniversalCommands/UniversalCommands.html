1. Git操作指令大全

1. 在操作之间必须设置SSH将本地仓库与远程仓库关联,这样才能进行push等操作, 设置SSH:

看本机是否存在SSH keys: $cd ~/.ssh#

新建:$ssh-keygen -t rsa -C "your_email@example.com" 设置密码一直回车就可以 不设置密码

根据路径:Your public key has been saved in /c/Users/you/.ssh/id_rsa.pub. 找到密码, 在GitHub设置相关的地方填好。


2. 本地已创建的创库与远程仓库设置关联:

在本地仓库下运行:

$ git remote add 远程仓库名 Github仓库链接

把本地库的所有内容推送到远程库上:

$ git push -u 远程仓库名 master


3. 从远程库克隆

通常情况下是先建远程库,在通过克隆的方式到本地:

cd到本地仓库要从存储的位置,如桌面: $ git clone 远程仓库链接

$ ls 查看克隆下来的东西

本地仓库添加需要的文件: $ git add readme.txt 或者添加所有 $git add . 添加失败可以使用管理者权限$ sudo git add . 进行尝试

提交到本地仓库(不需要网络):$ git commit -m "注释"

提交到远程仓库: $ git push


4. 创建与合并分支

查看分支：$ git branch (绿色的*显示当前操作的分支)

创建分支：$ git branch 分支名称

切换分支：$ git checkout 分支名称

创建+切换分支：$ git checkout -b 分支名称

在分支中进行增删改,添加到分支本地仓库:$ git add . $ git commit -m "注释"

合并某分支到当前分支：$ git merge 分支名称 (1. 合并之前先使用 git chekout master 切换到master; 2. 在合并之前不能分支内容不一致)

删除分支：$ git branch -d 分支名称(合并之后删除)

删除后可以使用$ git branch 查看分支

:q //退出编辑器

:q! //退出编辑器，且不保存

I //在当前行的开始处添加文本(非空字符的行首)

J //合并光标所在行及下一行为一行(依然在命令模式)

ls //显示当前目录的内容

pwd //显示当前目录的路径名

cd //改变当前目录

mkdir //创建一个目录

rmdir //删除一个目录

rm //删除文件或目录

cal //显示日历

telnet //远程登录
