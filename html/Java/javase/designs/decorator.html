设计模式之装饰模式

1. 装饰模式简介


        装饰模式（Decorator）:
            动态地给一个对象添加一些额外的职责，就增加功能来说，装饰模式比生成子类更灵活。
            优点： 扩展性强， 解耦合
            缺点： 维护工作量大

2. 实例

演示老王起床了，接下来要穿衣服，鞋子等

抽象实体类:


        //人穿衣服 鞋子 裤子等
        public abstract class Human {

            public abstract void display();
        }

具体实体类:


        //需要装饰的类 老王要穿衣服 鞋子等
        public class Laowang extends Human{

            @Override
            public void display() {
                System.out.println("老王起床了...");
            }
        }

抽象的装饰类:


        //抽象的装饰类
        public abstract class AbstractDecorator extends Human{

            private Human human;

            public AbstractDecorator(Human human){
                this.human = human;
            }

            @Override
            public void display() {

                if (human != null){
                    human.display();
                }
            }
        }

动作类：


        public class ChuanXiezi extends AbstractDecorator{

            public ChuanXiezi(Human human) {
                super(human);
            }

            @Override
            public void display() {
                super.display();
                System.out.println("穿鞋子");
            }
        }

        public class ChuanYifu extends AbstractDecorator{

            public ChuanYifu(Human human) {
                super(human);
            }

            @Override
            public void display() {
                super.display();
                System.out.println("穿衣服");
            }
        }

调用:


        Human laowang = new Laowang();
        //穿鞋子的动作
        laowang = new ChuanXiezi(laowang);
        //穿衣服的动作
        laowang = new ChuanYifu(laowang);

        laowang.display();
