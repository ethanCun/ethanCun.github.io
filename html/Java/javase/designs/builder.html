设计模式之建造者模式

1. 建造者模式简介


        建造者模式：
            1. 将一个复杂对象的创建与他的表示分离，使得同样的构建过程可以创建不同的表示;
            2. 对外部屏蔽了对象的创建细节 更容易扩展;

2. 实例

创建实体对象所有需要完成的动作:


        public class Itworker {

            public void wakeup(){
                System.out.println("起床");
            }
            public void wash(){
                System.out.println("洗脸");
            }
            public void breakfast(){
                System.out.println("吃早餐");
            }
        }

抽象建造者对应相关的动作:


        public abstract class IBuilder {

            public abstract void wakeup();
            public abstract void wash();
            public abstract void breakfast();
        }

建造者实现类，对外屏蔽实体对象的操作:


        public class DailyBuilder extends IBuilder{

            private Itworker itworker = new Itworker();

            @Override
            public void wakeup() {
                itworker.wakeup();
            }

            @Override
            public void wash() {
                itworker.wash();
            }

            @Override
            public void breakfast() {
                itworker.breakfast();
            }

            public Itworker getItworker() {
                return itworker;
            }

        }

封装操作建造者的构造类:


        public class DailyBuilderConstructor {

            private DailyBuilder dailyBuilder;

            public DailyBuilder getDailyBuilder() {
                return dailyBuilder;
            }

            public DailyBuilderConstructor(DailyBuilder dailyBuilder){
                this.dailyBuilder = dailyBuilder;
            }

            public void constructor(){

                this.dailyBuilder.wakeup();
                this.dailyBuilder.wash();
                this.dailyBuilder.breakfast();
            }
        }

调用:

        //对外部屏蔽了itworker的创建细节 更容易扩展
        DailyBuilder dailyBuilder = new DailyBuilder();
        //外部构造器
        new DailyBuilderConstructor(dailyBuilder).constructor();
