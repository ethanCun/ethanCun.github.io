设计模式之策略模式

1. 策略模式简介


        策略模式：
            策略模式是对算法的包装，把使用算法的责任和算法本身分隔开，委派给不同的对象管理;
            策略模式通常把一系列的算法包装到一系列的策略类里面，作为一个抽象策略类的子类。

         简单地说就是有一个管理类管理一个策略抽象类的所有实现类，可以在管理类中灵活操作策略实现;

2. 实例

策略抽象类:


        public interface IAction {

            void action();
        }

策略实现类:


        public class Say implements IAction{

            @Override
            public void action() {
                System.out.println("说话...");
            }
        }

        public class Sleep implements IAction{

            @Override
            public void action() {

                System.out.println("睡觉...");
            }
        }


        public class Eat implements IAction{

            @Override
            public void action() {

                System.out.println("吃饭...");
            }
        }

实体类:


        public class Human {

            //内部维护抽象动作类
            private IAction iAction;

            public Human(IAction iAction){
                this.iAction = iAction;
            }

            public void action(){
                this.iAction.action();
            }
        }

调用:


        IAction say = new Say();
        Human human = new Human(say);
        human.action();;

        IAction eat = new Eat();
        Human human1 = new Human(eat);
        human1.action();

        IAction sleep = new Sleep();
        Human human2 = new Human(sleep);
        human2.action();
