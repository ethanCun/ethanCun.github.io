设计模式之工厂模式与抽象工厂模式

1. 工厂模式简介


        1. 简单工厂模式又叫静态方法模式（因为工厂类定义了一个静态方法）;
        2. 将“类实例化的操作”与“使用对象的操作”分开，让使用者不用知道具体参数就可以实例化出所需要的“产品”类，
            从而避免了在客户端代码中显式指定，实现了解耦。

2. 实例:

产品信息抽象类：


        public abstract class IProduct {

            //产品信息
            public abstract void productInfo();
        }

具体产品实现类:


        public class Apple extends IProduct {

            @Override
            public void productInfo() {

                System.out.println("apple");
            }
        }


        public class Orange extends IProduct {

            @Override
            public void productInfo() {
                System.out.println("orange");
            }
        }

Product枚举:


        public enum Product {

            APPLE("apple"), ORANGE("orange"),BALALA("balala");

            private String name;

            Product(String name){
                this.name = name;
            }
        }

工厂类:


        public class Factory {

            //工厂模式的静态方法
            public static IProduct ProductFactory(Product product){

                switch (product){
                    case APPLE:
                       return new Apple();
                    case ORANGE:
                        return new Orange();
                }

                return null;
            }
        }

调用:


        //根据外部枚举通过工厂的静态方法直接生成对应的产品， 直接调用产品的方法, 解耦合
        Factory.ProductFactory(Product.ORANGE).productInfo();

3. 常见工厂类与抽象工厂类


        ResourceBundle.getBundle();
        Calendar.getInstance();
        ClassLoader.getSystemClassLoader()

        上面的工厂类还可以作为一个抽象的工厂类， 衍生出很多子工厂类， 这样就成了抽象工厂类模式
